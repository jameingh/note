## Java规则

- 检察事务是否存在一致性问题
- 检察入参是否合法
- 未使用 MyBatis-Plus 逻辑删除能力
- 使用 MyBatis-Plus 的 `BaseMapper`，无需编写自定义 SQL 或 XML 文件
- 检察逻辑删除是否和唯一约束冲突
	- 如果冲突，使用部分唯一索引：仅对未删除的记录生效，避免逻辑删除后重新绑定冲突 
- 不要忘记导入依赖(包括 pom.xml 和文件中的 import 语句)
- 每次确认修改代码后，同步修改相关设计文档
- 添加 Flyway 迁移脚本时，添加的sql语句要是幂等的，可以多次执行不报错
- sql 使用 postgresql 方言
- 批量查询列表时，一次性批量查询所有关联信息，避免 N+1 问题
- 更新数据时，仅当入参中组织字段非 null 时才更新对应关联，避免数据丢失
- 使用 Java17 特性
- 使用 Lombok 特性
- 创建测试用例，并运行直到通过
- 如果要写工具类，避免使用 @PostConstruct 将 Bean 赋值给 static 字段供全局调用。这会破坏 IoC 契约、引发类加载顺序问题、导致单元测试灾难。如何写，根据是否依赖 Spring 容器管理的 Bean
	- 纯工具类（无状态、仅依赖 JDK/第三方库）→ 静态类 + 私有构造器（最简）
    - 业务辅助类（需调用 @Service、@Repository、@ConfigurationProperties 等）→ @Component 实例类 + 依赖注入

## 如何避免追涨杀跌？

避免追涨杀跌，核心是建立规则来对抗情绪。你可以从这几方面入手：

1.  **制定交易纪律**：买入前就设定好条件，比如市盈率低于历史30%分位时才考虑买入。卖出也一样，股价高于价值10%就分批卖出。用机制替代直觉判断。

2.  **采用有纪律的投资策略**：
    -   **定投**：固定时间投入固定金额。市场涨时买得少，跌时买得多，自动拉低成本。
    -   **网格交易**：将资金分成多份，价格每下跌一定幅度买入一份，上涨一定幅度卖出一份，把波动变成收益。

3.  **做好仓位管理**：始终保持一定比例的现金或低风险资产（如30%）。手里有预备队，大跌时才有能力买入，而不是被迫卖出。

4.  **训练逆向思维**：
    -   当看到人人都在讨论“暴富”时，提醒自己风险在积聚。
    -   当市场悲观到没人敢谈股票时，可以开始乐观地寻找机会。

5.  **拉长决策周期**：想交易时，强迫自己再研究24小时。多看看**周线图**和**月线图**，减少分时图的干扰，避免被短期波动裹挟。

6.  **耐心等待“击球区”**：把机会想象成棒球比赛。如果没把握，可以一直“不挥棒”，只在自己最有信心的位置（如价格明显低于价值时）才出手。

7.  **记录决策日记**：每次操作后，写下当时的理由和情绪（如“怕错过”、“很恐慌”）。过段时间复盘，你会更清楚地看到情绪是如何影响判断的。

真正有效的投资往往是反人性的。如果想深入了解，我可以为你详细讲解**定投**与**网格交易**这两种具体方法，看哪种更适合你。

## 两种方法都讲讲
好的，我们来详细拆解一下**基金/股票定投**和**网格交易**这两种规避追涨杀跌的具体方法。

---

## 一、定投（定期定额投资）

### 1. 核心原理
固定时间（如每月8日）、固定金额（如2000元），买入同一个标的（如某指数基金或股票）。  
价格高时，固定金额买到的份额少；价格低时，买到的份额多。长期下来，持仓成本被自动拉低，无需预测市场。

### 2. 操作步骤
- **选标的**：长期趋势向上的宽基指数（如沪深300、标普500）或股价波动大但基本面不差的股票。不推荐单边下跌的垃圾股。
- **定周期**：按周、双周或月，周期越短平滑效果越好，但太频繁（如每天）意义不大。
- **定金额**：用不影响日常生活的闲钱，确保市场跌2～3年还能坚持。
- **设止盈（非必须）**：设定目标收益率（如20%、30%），达到后分批卖出，避免坐过山车。

### 3. 优点
- 极度简单，可自动化（银行、券商、基金App都能设置）。
- 几乎没有情绪消耗，越跌越买入，自然摆脱“追涨杀跌”。
- 资金门槛低，几十元起投。

### 4. 缺点
- 对“单边长期下跌”的标的无效（如果指数十年不涨，定投也会亏损）。
- 需要较长时间才能体现摊低成本的效果（通常3年以上）。
- 如果牛市很快来到，定投积累的份额较少，总收益不如一次性买入（但没人能精准抄底）。

### 5. 适合人群
- 每月有稳定现金流的工薪族。
- 不想盯盘、承认自己无法择时的投资者。
- 长期目标（养老、子女教育等）。

## 二、网格交易

### 1. 核心原理
把价格波动看作“一张网”：预先设定一个价格中枢（比如当前价10元），然后向上、向下每隔一定间距（如5%）挂买入和卖出单。  
- 下跌一格 → 买入一份  
- 上涨一格 → 卖出之前买入的那一份  
本质是把震荡行情中的涨跌都转化为利润，每个网眼完成一次低买高卖。

### 2. 完整设置步骤（以某股票或ETF为例）

1.  **选择标的**：必须是**波动大但长期不死的品种**，比如行业ETF、可转债、高波动蓝筹股。不能是单边上涨或单边下跌的股票（单边上涨会卖飞，单边下跌会不断买入被套）。
2.  **确定价格区间**：参考近6～12个月的高低点，设定一个底和顶（如12元 ~ 8元），超出区间就停止交易。
3.  **确定网格间距**：间距过大→交易次数少，利润薄；间距过小→频繁交易，手续费侵蚀利润。通常取3%～8%，波动大的品种间距可以大些。
4.  **计算每份金额**：总计划投入资金 ÷ 网格数量。例如总资金10万，分为10格，每格1万元。
5.  **挂单**：
    -   在当前价下方每隔一个间距挂一格**买入单**。
    -   在每一档买入价上方（买入价×（1+间距））位置挂对应的**卖出单**。
6.  **循环**：每成交一笔卖出，立刻在该卖出价下方一个间距的位置重新挂一个买入单；每成交一笔买入，立刻在该买入价上方一个间距的位置挂一个卖出单。自动低吸高抛。

### 3. 举例说明
- 股票现价10元，间距5%，每格1000股。
- 初始挂单：
  - 9.5元买入1000股，同时挂10元卖出这1000股。
  - 9.0元买入1000股，同时挂9.5元卖出。
  - 8.5元买入1000股，同时挂9.0元卖出。
- 价格跌到9.5元，触发买入。此时你持有1000股，并立即在10元挂1个卖单。
- 价格反弹到10元，触发卖出，获利500元（忽略手续费）。然后立即在9.5元处重新挂1个买单。
- 如此反复，只要价格在8.5~10.5元之间来回波动，每次震荡都产生利润。

### 4. 优点
- 在震荡市中能持续赚钱，克服追涨杀跌的本能（波动越大越开心）。
- 机械化操作，情绪干扰小。
- 可利用免费条件单自动执行（多数券商支持）。

### 5. 缺点
- **大牛市会卖飞**：价格突破网格上沿后，手里已经没有筹码，踏空主升浪。
- **大熊市被套牢**：价格跌破网格下沿，子弹打完，变成被动长期持股。
- 需要较频繁操作，交易成本累积需要注意。
- 对资金管理要求高（必须预先分好格子，中途不能随意增减）。

### 6. 适合人群
- 有一定交易经验、能接受“卖飞”和“套牢”两种后果。
- 判断市场未来一段时间是区间震荡（不是单边大趋势）。
- 有精力维护条件单或使用自动网格工具。

## 三、两者对比与组合使用

| 维度 | 定投 | 网格 |
|------|------|------|
| 市场适应性 | 长期向上（慢牛、震荡向上） | 震荡（牛皮市、窄幅波动） |
| 在单边大牛市的收益 | 不错，不会卖飞 | 很差，很快清仓踏空 |
| 在单边大熊市的亏损 | 会浮亏，但坚持定投可摊低成本 | 会重仓被套，资金锁死 |
| 资金需求 | 每月小额闲钱 | 一次性预留全部资金 |
| 自动化程度 | 极简单，全自动 | 需要条件单或定时检查 |
| 心理压力 | 几乎为零 | 中度（看到卖飞或套牢容易手痒干预） |

**一个常用的组合策略**：  
- 大部分存量资金做**网格交易**，赚取震荡市的波动收益。  
- 每月新增的现金（工资等）做**定投**，保证在长期上涨中不踏空。  
- 当市场突破网格上沿（大牛市启动信号），暂停网格，将新增资金全部转为定投；当市场跌破网格下沿（恐慌熊市），暂停定投，专注等待网格触底信号。

这样既利用了定投的长期复利，又用网格在无聊行情中“剪羊毛”，还能规避单边市的各种陷阱。
