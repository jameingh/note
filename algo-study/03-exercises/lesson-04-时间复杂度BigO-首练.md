## 一、直觉（先写）

### 1) 为什么分析算法时常说「基本操作次数」而不是真实运行毫秒数？

你的作答：
运行毫秒依赖硬件，是非中立的描述，基本操作次数则相反。


### 2) 「最坏情况」为什么常作为默认讨论对象？

你的作答：

最坏情况用来评估最倒霉时会慢到什么程度，是兜底保证。


### 3) 下面这些增长趋势，按「快」到「慢」排序（从小到大，用你自己的话写即可，例如「常数级、对数级……」）

关系：\(O(1)\)、\(O(\log n)\)、\(O(n)\)、\(O(n \log n)\)、\(O(n^2)\)

你的作答：

O(1)、O(logn)、O(n)、O(nlogn)、O(n^2)

## 二、两个小练习（写过程，不直接跳答案）

下面设 \(n\) 为「问题规模」（例如数组长度）。

### 练习 A：单层循环

伪代码：

```
for i in range(n):
    做一点点常数工作
```

请你：用自然语言描述「循环体执行多少次」「与 n 的关系」。

你的作答：
n越大，循环体内执行次数越多。正相关
当n为2时，循环体执行大约n次，线性相关。


### 练习 B：二层循环（典型嵌套）

伪代码：

```
for i in range(n):
    for j in range(n):
        做一点点常数工作
```

请你：用自然语言描述「内层那行『常数工作』总共执行多少次」。

你的作答：
总共执行了n*n次

## 四、卡住时提示（卡住再看）

提示 1：Big-O 描述的是「随 n 增长时，工作量如何增长的趋势」。

提示 2：嵌套循环时，常问：最内层那一步被执行的总次数是多少？

提示 3：\(O(\log n)\) 常与「每次把规模砍半」联系在一起，例如二分查找（本课只需先记住直觉）。
