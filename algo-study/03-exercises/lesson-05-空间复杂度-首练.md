## 二、场景判断（先判断，再写一句理由）

设输入数组长度为 `n`，判断下面代码的空间复杂度趋势（只需写“常数级/线性级”等自然语言）：

### A

```python
sum = 0
for x in arr:
    sum += x
return sum
```

你的判断与理由：

线性级。

我重新判断为常数级，因为sun和x不会随着数组长度增长而增长，这两个都是临时变量，下次循环接着用。不会额外占用空间。

### B

```python
out = []
for x in arr:
    out.append(x * 2)
return out
```

你的判断与理由：

线性级。out数组长度随着arr的长度增长而增长。

### C

```python
def f(n):
    if n <= 1:
        return 1
    return f(n-1) + 1
```

你的判断与理由（提示：想想调用栈）：

常数级。因为递归函数会创建一个临时变量，所以空间复杂度是O(1)。

我重新判断为线性级，因为递归深度会带来额外的栈帧的空间占用。空间占用是随着n的增加而增加的。

一句话总结：循环代码看空间时，先看额外变量个数；递归代码看空间时，先看递归深度。
